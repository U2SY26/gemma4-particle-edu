## Production Checklist

### Pre-Recording

- [ ] Ollama installed and running with `gemma4:latest` model loaded
- [ ] App running locally at `localhost:3000` (or deployed URL)
- [ ] OBS Studio configured: 1080p, 60fps, browser source or window capture
- [ ] Chrome in full-screen mode, no bookmarks bar, no extensions visible
- [ ] Dark mode OS theme to match app aesthetic
- [ ] Terminal with clean profile (no personal info visible)
- [ ] Korean and English input methods ready
- [ ] Test all three scenarios before recording (dry run)

### Recording Order

1. Record all app demos (Scenes 2.1, 2.3, 3.x) in one session
2. Record terminal/Ollama shots (Scene 2.3) separately
3. Create motion graphics slides (Scenes 1.x, 2.2, 2.4, 4.x) in video editor
4. Record voiceover narration after video edit is assembled

### Post-Production

- [ ] Add background music (start at 0:00, fade out at 3:00)
- [ ] Suggested tracks: YouTube Audio Library -- search "upbeat technology" or "educational"
- [ ] Volume: music at 15-20% during narration, 40% during transitions
- [ ] Add text overlays and annotations per scene descriptions
- [ ] Add subtle zoom/pan to static screen recordings for visual interest
- [ ] Color grade: slightly boost contrast and saturation on neon elements
- [ ] Add neon glow post-effects to transition moments
- [ ] Export: H.264, 1080p, 30fps minimum (60fps preferred for smooth simulations)
- [ ] Upload to YouTube as Public or Unlisted (must be accessible without login)
- [ ] Add to Kaggle submission Media Gallery

### Narration Guidelines

- **Tone**: Confident but warm. Educational but exciting.
- **Pace**: Moderate speed. Pause 0.5s between major points.
- **Language**: English primary narration. Korean prompts shown on screen with subtitles.
- **Total narration time**: Approximately 2 minutes 20 seconds (remaining time is visual-only).
